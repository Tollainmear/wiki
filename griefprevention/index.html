首页

这些内容很重要 - 升级到版本 4.0.0+ 之前请务必阅读

这是一次变化非常大的更新，所以强烈建议您升级前备份所有数据

如果你是从 4.0.0 之前的版本升级的, 强烈建议您跳过下列内容并确保config中的 'wilderness-cuboids' 设置成了 false ，这将在2D领地的基础上支持3D领地

如果你已经用上了版本 4.0.0 并想把领地系统从 3D 迁移至 2D , 请在config中把 'migration-2d-rate' 设置成和升级至 4.0.0 时一样的值

迁移

如果你真的想启用 wilderness-cuboids ，你需要充分知晓这需要一次完全的改变，领地内方块的处理方式由计算高度变为了计算方块本身，这样做是为了让创建3D领地计费更科学，也就是说当玩家创建 10x10x10 的领地以及 10x10x256 的领地时，他不需要支付一样的钱，在设置这个属性时请务必完全测试一遍

新版本更新添加了3D领地的支持，如果你想让玩家创建3D领地，请在config中把 'wilderness-cuboids' 设置成 true 这将让创建领地通过方块计数来计费 3D领地系统如果启用了 下列内容将会改变 :

  • 每个方块都会花费1领地方块的价格
  • Y 会被算作领地方块计价的因素
  • 2D 领地必须从天顶到地底
  • 任何天顶到地底之间的领地都将会是 3D.
  • 一个 2D 的区块 (16x16x256) 领地将算作 65,536 个方块，而不是 256 个方块，因为 Y 轴也会被计算进去

想要迁移已经存在的方块玩家数据，请编辑插件的 config 在玩家数据项目下, 把 'migration-3d-rate' 设置为一个你想让 原本2D面积乘以的一个数，建议默认设置为 256, 当然你想设置什么数都可以 设置成 0 将重置所有方块数据 迁移只会运行一次，它将标出所有改变过的数据

如果你想再次运行迁移 只需将config中的 'reset-migrations' 设置为 true ，如果你不做这一步就想再次运行迁移，将什么都不会发生，因为所有玩家数据都已经被标记为迁移过的了

GriefPrevention Wiki

适合 Sponge 服务器的插件. 名字与适合 Bukkit 服务器的插件相同，但是实际上不一样

  • Sponge Forums
  • Official Downloads
  • Discord Server - for support, questions, or general GriefPrevention related talk.

一个绝佳的一站式的防熊方案，保护你的服务器以及世界 你可以将它完全整合进你的权限插件 强烈推荐 LuckPerms 因为其它权限插件可能会带来一些bug

插件拥有一个直观的用户界面，可用于编辑领地属性或者设置管理员领地，易读易用

这个插件既可以在纯净服务器使用也可以在 Forge服务器使用 (最新版的 SpongeVanilla/SpongeForge (b1878+))

注意: 不再支持 PermissionEx ，因为这插件已经死了，没救了
